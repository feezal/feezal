# U57 — Theme selector: richer swatch (bg chip + role dots + accent) ✅ implemented

*Archived roadmap item — Editor UX. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**Delivered (07/2026):** `sampleThemeColors()` now samples seven canonical roles per theme (`--primary-background-color`, `--secondary-background-color`, `--primary-color`, `--primary-text-color`, `--secondary-text-color`, `--divider-color`, `--accent-color`) into a keyed record; the shared `<feezal-theme-select>` renders the compound swatch — a two-tone background chip carrying, all inside it, three larger role dots, a divider rule, and a smaller accent dot — in both mounts (site sidebar + per-view inspector). Legacy `[bg,text,text2]` arrays coerce via `toSwatchRecord`, and `fillSwatch` backfills any role a theme omits (plus the built-in default and not-yet-sampled placeholder) so the chip never has an empty slot. `DEFAULT_SWATCHES`/`PLACEHOLDER_SWATCHES` became role records; the themes sidebar’s pass-through `{default: DEFAULT_SWATCHES, ...}` merge was unaffected. `theme-visual.test.js` baselines did NOT need regenerating (they render the raw site vars, not the picker swatch).


The theme picker previews each theme with **three flat circular dots** — `--primary-background-color`, `--primary-text-color`, `--secondary-text-color` (fallback `--divider-color`), sampled by `sampleThemeColors()` and rendered as `.swatch` circles ([feezal-theme-select.js:45-73](../www/src/feezal-theme-select.js#L45-L73), [:111-115](../www/src/feezal-theme-select.js#L111-L115), [:234-236](../www/src/feezal-theme-select.js#L234-L236)). Three dots convey too little: two of the three are text tones, the background is a single dot, and the accent colour — the thing that most distinguishes one theme from another at a glance — isn't shown at all.

**Wanted — a compound swatch:**

- A **chip** (rounded rectangle), split diagonally or vertically into two halves: left `--primary-background-color`, right `--secondary-background-color` — so the theme's surface contrast reads immediately.
- **Three dots sitting on the chip**: `--primary-color`, `--primary-text-color`, `--secondary-text-color` (the accent/brand plus the two text tones, shown against the real surfaces behind them).
- A thin **divider** drawn in `--divider-color` (doubles as a live sample of that variable).
- Right of the divider, a **small dot** in `--accent-color`.

Net: one preview samples **seven** canonical theme variables (`--primary-background-color`, `--secondary-background-color`, `--primary-color`, `--primary-text-color`, `--secondary-text-color`, `--divider-color`, `--accent-color`) instead of three, and shows text/accent legibility *against the actual backgrounds*.

**Implementation notes:**

- Extend `sampleThemeColors()` to collect the seven vars (it already samples by temporarily applying each class to `feezal.site`; add the extra `getPropertyValue` reads). The return shape changes from a flat `[c1,c2,c3]` array to a keyed record per class — update `_swatches()`/the render and the `DEFAULT_SWATCHES` / `PLACEHOLDER_SWATCHES` constants to the new shape.
- **One component, both mounts (U53/E106):** `feezal-theme-select` is shared by the themes sidebar (site theme) and the view inspector (per-view `theme`), and the theme cards in the themes sidebar itself may want the same richer preview — keep the swatch a single building block so all consumers stay in sync by construction.
- Keep it compact: the trigger swatch is ~13 px tall today and sits in a one-line row; the chip + dots + divider + accent must still fit that height in the closed trigger and read cleanly in the taller dropdown rows.
- The default (built-in) theme and the not-yet-sampled placeholder need sensible neutral values for all seven roles.

**Ships with:** the `theme-visual.test.js` swatch-screenshot baselines regenerate; a unit test that `sampleThemeColors()` returns the seven roles per class; TESTING.md note (the theme picker preview shows the bg chip + primary/text dots + divider + accent dot, in both the themes sidebar and the view inspector, light and dark).
