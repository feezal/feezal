# U37 — Welcome wizard / first-run onboarding tour ✅ implemented

*Archived roadmap item — Editor UX. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

> **Status: ✅ implemented (July 2026)** — `www/src/feezal-welcome-tour.js`, hand-rolled spotlight overlay (no tour library: editor-only, nothing reaches the viewer/export bundle, and the cutout tracks targets inside the editor's shadow DOM which selector-based libraries can't reach). Dimmed backdrop via a positioned cutout div whose 200vmax box-shadow paints the dim; explanation card with Next / Back / Skip and progress dots; cutout animates between targets and re-measures on resize + a coarse interval.
>
> **Steps (as specced):** palette → canvas → inspector (switches the sidebar to Inspector itself) → Deploy/▾ View → MQTT broker setup (opens Site Settings, interactive — the existing status indicator gives feedback) → hands-on: drag a `basic-template`, wire `subscribe` (B28 autocomplete helps) + template content `${msg.payload} °C`, deploy & view. The hands-on steps advance **event-driven** via MutationObservers (element landed → topic + template content set) with Next as manual override. Non-interactive steps place a click-catcher so stray clicks can't derail the tour; interactive steps make the overlay purely visual.
>
> **Trigger & persistence:** auto-starts only when the seen-flag is unset AND no view carries any element (fresh install); Skip/Done set the flag. Re-launchable from Editor Settings → Help → "Show welcome tour". Flag was initially localStorage-only; **corrected 07/2026 (U41 follow-up)**: the seen-flag now persists server-side in <dataDir>/editor.json via /api/editor/prefs (authoritative when reachable — deleting the data dir re-triggers the tour, exactly the retest case that exposed the localStorage-only gap), with localStorage as serverless fallback. Unit tests (trigger gating) + browser tests (step machinery, sidebar switching, click-blocking, hands-on advance, persistence).
