# E136 — Metro tile backsides: redesign for touch — bigger targets, calmer layout, use the space ✅ done

*Archived roadmap item — Element Ecosystem. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Implemented (07/2026)** — leaning INTO the Metro language as decided (touch-first flat surfaces, full-width segments, typography-led, one accent):

- **Shared layer (`MetroTileBase`, family-wide):** `.mbtn` becomes a Metro flat segment — borderless quiet fill (`color-mix` of the text colour), **min-height 36 px** (half a 70 px grid unit), invert on hover/active, **pressed-scale feedback** (the classic press effect, matching the flip vocabulary). Sliders: 6 px rails, 18×28 px thumbs, **32 px hit area** (hit larger than the visual), both engines. `.back-content` gets real insets/gaps and the size-adaptive layout: **4x2-and-wider lays direct children out as side-by-side half-width zones** (`@container (min-aspect-ratio: 3/2) and (min-width: 220px)`), **4x4 gains breathing room** (`min 260×260` → bigger insets/gaps), 2x2 stacks. Bigger flip-affordance hit area. All type sizes ride the E129 tokens — no new hardcodes.
- **metro-climate (the worst offender):** the setpoint stepper is now the WP7-volume-style pair of **giant flat +/− halves (≥48 px)** with the setpoint in the E129 display type between them, edge to edge; mode chips are full-width flat segments (`flex: 1 1 auto`, ≥36 px, unit-token type). At 4x2 the stepper zone sits left, the mode segments right (the generic zone split).
- **metro-media:** transport buttons stretch as segments. light/wled/cover/sensor/number backs inherit the shared layer (segments, sliders, layout) with no per-element surgery — fronts untouched, MQTT contracts/attributes/flip machinery untouched (visual/layout only, as scoped).

metro-tile 3.0.3, metro-climate 3.0.11, metro-media 3.0.2 (the other family members inherit via the shared base — no file change, no bump). Browser tests: stacked-vs-side-by-side per tile size (both directions), ≥48/≥36 px targets, pressed-state declaration, 32 px slider hit area. TESTING.md metro row extended with the explicit wall-tablet thumb checks. Theme-visual screenshot refresh rides the next visual-test update.

Original item (abridged): the flip-to backs don't hold up — buttons too small, space wasted (especially 4x2), bad on touch. Design brief: stay true to Metro (touch-first, flat, full-width hit areas, sharp corners, typography-led, one accent) — targets ≥ half a grid unit, size-adaptive back layouts per tile size (4x2 side-by-side with WP7-volume-style giant stepper halves, 2x2 stacks, 4x4 breathing room), calmer hierarchy on the E129 tokens, visible pressed states, thumb-sized sliders. Backs only; contracts untouched.
