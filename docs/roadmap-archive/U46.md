# U46 — Clippy easter egg in the help popup ✅ implemented

*Archived roadmap item — Editor UX. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Implemented (07/2026)** — an off-by-default **Editor Settings → Help** toggle ("Assistant in the keyboard-shortcuts popup", persisted in `localStorage` key `feezal-clippy`). With it on, **both** `?` / Ctrl+I keyboard-shortcuts popups — the **canvas-mode** one (in `feezal-sidebar-inspector`) and the **source-mode** one (in `feezal-app-editor`) — render an **original self-drawn** paperclip + speech bubble BELOW the shortcut table (in-flow, never covering it), gently bobbing; the bob is disabled under `prefers-reduced-motion`. Light/dark styled, no Microsoft asset, nothing from a CDN (A25). The markup + styles live in one shared module (`feezal-clippy.js`) so the two popups can't drift; dark mode in the nested inspector is piped via `--feezal-clippy-*` custom properties. Original item:


Optional, off-by-default **Editor Settings** toggle that adds a paperclip assistant — a deliberate homage to the late-90s Microsoft Office assistant — to the help popup. Purely a joke feature.

**Constraints:** must be **opt-in**, must not interfere with the actual help content, and the artwork must be **self-drawn/self-hosted** (an original paperclip, not Microsoft's asset — trademark/copyright) and bundled locally per A25 ✅ (no CDN — implemented 07/2026). Respect `prefers-reduced-motion` if it animates.

**Relates:** B43 ✅ (help popup bug — was the blocker, fixed 07/2026), A25 ✅ (self-hosted assets — implemented 07/2026), U37 ✅ (welcome wizard — the other "friendly onboarding" surface).
