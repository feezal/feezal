# N37 — Pause subscriptions of hidden views (bandwidth saver) ✅ done *(view-level MVP)*

*Archived roadmap item — Near-term Improvements. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Implemented (07/2026)** — the decided view-level MVP, viewer only:

- **`FeezalVisibility`** ([feezal-visibility.js](../../www/src/feezal-visibility.js)) — the central controller, instantiated by `feezal-app-viewer` (the editor never does): observes the site's active view, applies the per-view **grace timer** before pausing (default 30 s, resume always immediate), and exposes `isPaused(el)` as the connectedCallback precondition.
- **`FeezalElement`** (3.0.6) — `pauseSubscriptions()` (uniform teardown: primary `_subscriptions`, N31 availability, conditions engine) and `resumeSubscriptions()` (a **detach/re-attach reconnect cycle** — the one wiring path every element, incl. manual-wiring device cards, implements correctly; retained values repaint instantly from the B40 cache replay). The pause precondition is enforced in **`addSubscription` itself** — device cards wire in their own connectedCallback after the base guard returned, so the gate lives where it is universal.
- **Config as serialized site attributes** (playlist/presence precedent): `<feezal-site pause-hidden-subscriptions pause-grace-seconds="30">` (Site Settings → Viewer → *Bandwidth* switch + grace input) and the per-view tri-state `pause-subscriptions` = `inherit | always | never` (view inspector select). Default **off** — fully backwards compatible; `always` pauses even when the site default is off.
- **`never` / warm-cache detail verified:** the B40 cache refreshes on live messages for retained-origin topics while any subscriber exists, so a `never` view keeps the cache warm for later embedded clones; pure command topics are deliberately never cached/replayed — no cache change was needed.
- Hidden originals of embedded views pause under `inherit` (their clones subscribe/unsubscribe on their own lifecycle, unchanged); elements stamped into an already-paused context do not subscribe (precondition) and wire on resume.

**Later tiers (explicitly deferred as decided):** element-level pausing (IntersectionObserver), page-level pausing on `document.visibilityState`, and the feezal/socket backend per-client subscription-list verification noted in the item.

Browser tests: grace pause + instant resume, no-churn back-and-forth, `never`, `always`-over-site-off, and the stamped-into-paused precondition. TESTING.md section added.

Original item (abridged): every element of every view subscribes on load and stays subscribed — hidden views consume bandwidth forever. Decided: view-level granularity, site default + per-view tri-state (`never` = escape hatch for non-retained data), viewer only, configurable grace (default 30 s), central visibility controller, resume via the element's normal wiring with B40 cache repaint.

**Relates:** B40 ✅ (retained cache/replay — the resume mechanism), N31 (availability pauses too), layout-app / dialog-view / layout-view (embedding paths — clone lifecycle already correct), N12/N13 (the other bandwidth items), A18 (kiosk/wall tablets — the main beneficiary), U51 ✅ (per-view settings precedent).
