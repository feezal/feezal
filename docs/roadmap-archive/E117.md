# E117 — `publish-local` on every publishing element ✅ done

*Archived roadmap item — Element Ecosystem. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Completed (07/2026)** — both tranches shipped; the refined scope (navigation + dialog-opening only, NOT a blanket rollout) is final.

- **Buttons tranche** (07/2026): shared `publishLocalAttribute` descriptor in `@feezal/feezal-element` 3.0.3; all six button elements (`material-button`/`icon-button`/`fab`, `glass-button`, `carbon-button`, `paper-button`) thread `{local: this.publishLocal}` into `pub()`.
- **Navigation/tab-bar tranche** (07/2026, completing the item): `material-navbar` 3.0.2, `tui-menu` 3.0.1, `metro-tile` 3.0.1 (the generic action tile only — not the whole Metro family), `material-chip` 3.0.1 — shared descriptor + option threading; `paper-tabs` 3.0.1 (Polymer) converged its legacy `publishLocal` on the shared descriptor wording (inline copy + explicit `value: false`). `basic-navigation` confirmed publish-free — no attribute (it switches views locally already). **Bonus fix surfaced by the tests:** material-navbar never declared `publish` as a Lit property, so the attribute from saved markup silently never published in the viewer — declared now.
- Browser tests cover option threading + the shared descriptor for all four Lit elements (`test-browser/feezal-publish-local-navigation.test.js`); TESTING.md gained the navigation-tranche row.

Original item:

**Scope refined (07/2026): navigation and dialog-opening only — NOT a blanket rollout.** `publish-local` exists for page-local UI wiring (open a dialog, drive a view/tab switch); sliders, checkboxes, selects and the complex device elements publish *device state* that belongs on the broker — they explicitly do **not** get the attribute.

**Remaining (the navigation/tab-bar tranche):** spread `publishLocalAttribute` the same way as the buttons (Lit: import + property + constructor default + pub options; Polymer: inline copy) to every element that publishes as a navigation/selection act: `material-navbar`, `paper-tabs` (converge its legacy `publishLocal` on the shared descriptor), `tui-menu`, `metro-tile`, `material-chip`; `basic-navigation` needs checking, likely nothing.

**Relates addition:** the dialog elements' *openers* are the consumers — a `publish-local` button/navbar entry triggering a `*-dialog`'s subscribe topic is the canonical use (E49 ✅ semantics).

Original motivation: a button that opens a dialog had to publish its trigger to the **broker** and wait for it to come back. That round-trip is pointless for pure UI wiring (open a dialog, switch a view, drive another element on the same page), puts UI chatter on the broker, and fails outright when the connection is down — while `FeezalConnection.pub()` supported `{local: true}` all along.

**Relates:** E49 ✅ (page-local publish semantics), **E114** (family parity — the mechanism to land the remainder consistently), E118 ✅ (the sibling attribute-gap item, archived), N24 (per-client control topics — the deliberately *non*-local counterpart).
