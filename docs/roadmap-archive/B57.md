# B57 — glass-climate: Homematic mode shown as a raw number (and Auto's mode text hidden entirely) ✅ fixed

*Archived roadmap item — Bugs. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Fixed (07/2026)** — the glass front state line now resolves the read-back through `_activeModeEntry` and renders the entry's **label** (`• Manu`, `• Auto` incl. mode 0, `• Off` via the B53 sentinel — which also suppresses the `→ 4.5°` target), guarded without truthiness; an unknown read-back still shows the raw value as the fallback. Metro was checked for the sibling defect: its front shows no mode text (chips only, like material's) — nothing to fix there. Browser-tested (label vs raw number, mode-0 visibility, raw fallback). Part of the B53–B58 climate fix wave.

Original item:

The glass climate card's state line shows the Homematic mode **numerically** — `→ 21,5 • 1` instead of `→ 21,5 • Manu`.

**Cause — one template expression, two defects:**

```js
${this._mode ? ` • ${this._mode}` : ''}
```

1. **Raw value instead of label:** it renders `this._mode` — the wire value — directly. HA string modes ("heat") happen to read acceptably; Homematic's numeric modes print `• 1`. The E102 active-entry resolver **already exists in this element** (`_activeModeEntry`, `match-setpoint-max`-aware) — the state line just doesn't use it. Render the resolved entry's **`label`** (`• Manu`, and correctly `• Off` once B53 lands, since the resolver is cap-aware).
2. **Falsy-zero (the B55 sibling):** `this._mode ? …` — Homematic **Auto is mode `0`**, so the mode suffix is hidden entirely while in Auto. Guard on the resolved entry instead of the raw value's truthiness.

**Also check while at it:** metro-climate's front/state text for the same raw-value rendering (material shows the mode via chips and is fine); and the fallback when NO entry matches the read-back (unknown wire value) — showing the raw value then is correct, just not as the primary path.

**Acceptance:** hm thermostat: state line shows `• Auto` / `• Manu` / `• Off` (labels, incl. mode 0); HA/z2m climates unchanged (string modes keep working, capitalisation via label as today in the popup); unknown mode values still show raw rather than nothing.

**Relates:** E102 ✅ (`_activeModeEntry` — the resolver the state line must use), **B55** (the same falsy-zero class in the same family — fix in one sweep), B53/B54/B56 (the climate fix wave), **E114** (check metro for the sibling defect).
