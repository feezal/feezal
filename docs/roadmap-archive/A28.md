# A28 — Configurable per-site CSP: "Security" tab in Site Settings ✅ done

*Archived roadmap item — Architecture & Infrastructure. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Implemented (07/2026)** — per the decided design:

- **Header builder** ([server/src/csp.js](../../server/src/csp.js)): per-aspect `{mode: all|self|hosts, hosts[]}` under `viewer.security.csp` (aspects: `images` = img+media, `frames`, `connect`, `scripts`, `styles`, `fonts`); absent config → the exact A25 baseline (fully backwards compatible). **Invariants enforced by the builder, not the UI:** script keeps `'unsafe-inline' 'unsafe-eval'`, style keeps `'unsafe-inline'`, font keeps `data:`, img/media keep `data: blob:`, `connect-src` always includes `'self' data:` (B51) **+ the broker origin** (auto-derived from the connection URI; bridged mqtt:// → covered by `'self'`); worker/base/object stay at baseline. Origins validated as `scheme://host[:port]` / `*.example.com`, no paths. Applied by the **viewerHandler to the HTML document only** — assets keep the global baseline; the **editor CSP stays fixed**.
- **Violation-driven suggestions in v1:** the viewer header carries a same-origin `report-uri /api/csp-report/<site>` (`report-uri` chosen — deprecated-but-universal); a public POST endpoint feeds a per-site in-memory **ring buffer** (cap 50, aggregated host+directive, local only per the A25 stance); a protected GET (`/api/sites/:name/csp-violations`) serves the Security tab.
- **Security tab** (`feezal-sidebar-viewer`, new tab beside Connection/Site/Viewer/Clients): human-labelled rows (*Images & cameras*, *Embedded pages (iframes)*, *Network connections*) + an **Advanced** disclosure for Scripts/Styles/Fonts with the prominent loosening warning; 3-state selects + origin chips with inline validation; locked 🔒 broker chip; **content-scan chips** ("found in your dashboard — allow") from the site markup; **Recent violations** list with one-click Allow and the changes-apply-on-deploy hint. Persisted through the normal deploy pipeline (`viewer.security`).
- **Exports: out of scope** as decided.

Unit tests (builder modes/invariants/broker/validation, ring buffer cap + isolation, both endpoints); TESTING.md section; user-guide §14 privacy addendum (per-site CSP + the local-only report endpoint). The tighten→violation→allow e2e remains a manual TESTING row (the unit layer covers the mechanics).
