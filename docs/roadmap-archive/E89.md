# E89 — Lottie animation element (`feezal-element-basic-lottie`) ✅ implemented

*Archived roadmap item — Element Ecosystem. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

Shipped 07/2026. A Basic element playing a Lottie animation with MQTT-driven playback. Introduced the shared package **`@feezal/feezal-lottie`** which owns the single memoised `import('lottie-web')` site, so lottie-web lands in one on-demand chunk (verified split, ~308 kB, absent from the base viewer entry) — reused by E39. Viewer-only lazy load gated on an effective src (a no-src element loads nothing); editor renders a static film-strip chip and never touches the lib. `map` (value → clip descriptor: segment seek / loop-speed override / src swap) is checked before the transport payloads; a src change destroys+recreates the instance; ResizeObserver → `resize()`; `destroy()` on disconnect; broken src → subtle placeholder. Attributes `src`/`subscribe`/`message-property`/`payload-play|pause|stop`/`map`/`autoplay`/`loop`/`speed`; `--feezal-lottie-background`. 12 browser tests + smoke; TESTING.md §6 entry.
