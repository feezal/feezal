# E134 — Circle design language: align the remaining device cards with light/climate/cover/switch ✅ done *(canon + sensor disc; per-element audit recorded)*

*Archived roadmap item — Element Ecosystem. Open items: [../ROADMAP.md](../ROADMAP.md) · Index: [README.md](README.md)*

**✅ Implemented (07/2026):**

- **The canon is written down** — `docs/element-spec.md` §5.2b "The Circle card canon": width-sized top-anchored circle, the shared `--feezal-*-track-width` 7 / `--feezal-*-knob-size` 10 tokens, rows-stack-below, 180×220 defaults (E123 precedent: new drops only), state-disc pattern with cqi sizing, canonical colours, chrome placement (unavail top-right, battery top-left, label bottom), and the explicit "not everything is a circle" rule.
- **Primary redesign shipped:** the Circle **Sensor** card (material-motion 3.0.4) renders hazard/generic classes as the **circle state disc** — centred type icon + state text inside a ring, accent while active, error-coloured for alarm classes, cqi-scaled (`container-type: inline-size` on the host). The four motion types deliberately keep their dedicated SVG visuals (audit decision below). Browser-tested.

**The audit (the per-element decision list the item asked for):**

| Element | Decision | Rationale |
|---|---|---|
| sensor (material-motion) | **disc** ✅ shipped | hazard/generic classes; motion types keep their SVG art — it already speaks the circle-adjacent language and is load-bearing for users |
| outlet ("Switch") | already canon | E121/E122 power disc IS the reference disc |
| light / climate / cover | already canon | B29/B37/E123 pinned the geometry |
| contact | **keep-but-align** | the window/door/garage SVG visuals (incl. tilt) are load-bearing information — a disc would lose them; alignment = tokens/proportions only |
| door-lock | **disc candidate — deferred** | natural state-disc fit, but deserves its own pass together with its Keymatic actions; not forced in this item |
| motion types of the sensor card | keep visuals | see above |
| camera, media-player, energy-flow, plant, tank, schedule, gauge-likes | **exempt** | legitimately non-circular; alignment = shared spacing/typography/theme tokens + 180×220-compatible proportions |

**Constraints honoured:** visual only — MQTT contracts, attributes and discovery untouched; saved sizes untouched; glass/metro out of scope (own languages; E114 parity applies within each family).

**Follow-up left open deliberately:** the door-lock disc pass (audit-flagged), and theme-visual screenshot refreshes ride the next visual-test update.
